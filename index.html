Key Projects

ODIGE — Energy Generation Analytics Platform

The project centred on building a reliable, cost-efficient data pipeline to aggregate energy generation data from wind and solar farms across multiple UK regions. The starting point was a thorough analysis of the Elexon API — the UK's official electricity market data provider — to understand available endpoints, data formats, and rate limits. That analysis directly shaped the ingestion strategy.

Ingestion & Lambda Design

A scheduled AWS Lambda function, triggered once daily via EventBridge, was chosen over a containerised ECS task deliberately: the job completes in under 15 minutes, runs once a day, and carries no persistent state — there was simply no justification for the overhead of a running container. The Lambda pulls data from Elexon, cleans and normalises it to reconcile inconsistencies between wind and solar farm formats, and writes both raw CSV and processed JSON to S3.

Storage & Query Layer

S3 was chosen as the primary storage layer — not a relational database — because the data is append-only, time-series, and grows continuously. S3 handles that access pattern at a fraction of the cost of RDS or DynamoDB, with effectively infinite scalability. AWS Glue was configured to automatically crawl the S3 buckets, infer schemas, and populate the Glue Data Catalog, transforming raw CSV and JSON files into structured tables. Athena then provided serverless SQL querying directly on S3 — charged per query scanned, which at this data volume was significantly cheaper than provisioning any persistent query infrastructure.

Visualisation — Two Audiences

AWS Managed Grafana was integrated directly with Athena to give engineering and operations teams near real-time dashboards covering energy generation trends, regional comparisons, and fault detection signals. Amazon QuickSight was connected separately for business stakeholders who required self-service reporting and scheduled email summaries. The distinction was intentional: Grafana excels at operational monitoring with live data refresh and metric alerting; QuickSight is better suited to business intelligence workflows where non-technical users need to explore data independently.

Britvic — IoT-Connected Flavoured Water Dispense System

Britvic is one of the UK's largest soft drinks companies and a flagship client at Green Custard. The project centred on an IoT-connected flavoured water dispense system, where a fleet of dispense units continuously stream operational data to the cloud — covering flavour recipes, cartridge status, dispense events, and error logs.

Hardware & Firmware

At the hardware level, I contributed bare-metal STM32 firmware for the RFID cartridge interface, handling low-level peripheral communication to identify and validate flavour cartridges inserted into the dispense units.

Edge Layer

The edge layer ran on Raspberry Pi with AWS Greengrass, managing MQTT communication between the physical dispense units and AWS IoT Core. Greengrass handled local message routing, buffering during connectivity gaps, and secure device-to-cloud synchronisation.

Cloud & Application Layer

Incoming data was routed into DynamoDB for structured dispense and user records, and S3 for error logs and telemetry archives. The application layer was built fully serverless — API Gateway and Cognito handled authentication and request routing, with Lambda functions managing data processing and business logic.

Infrastructure & CI/CD

The full infrastructure was defined as code using AWS CDK, with a CI/CD pipeline through Bitbucket, CodeBuild, and CodePipeline automating testing and deployment from development through to production.
